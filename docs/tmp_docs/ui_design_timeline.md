# 界面设计时间线与设计顺序

## 设计策略说明

本系统采用**渐进式设计策略**，分三个阶段完成所有界面设计：

1. **阶段一：用户端基础界面** - 设计顾客购物流程的核心界面
2. **阶段二：商家端管理界面** - 基于用户端界面，复用设计模式，添加商家管理功能
3. **阶段三：管理员端审核界面** - 基于商家端界面，复用设计模式，添加审核功能

**设计复用原则**：
- 商家端复用用户端的商品列表、商品详情、订单列表等页面布局
- 管理员端复用商家端的管理页面布局，添加审核功能模块
- 保持统一的视觉风格和交互模式

## 阶段一：用户端界面设计（19个页面）

### 设计顺序与页面关系

**1. 认证流程（3页）**
- 注册页 → 登录页 → 忘记密码页
- 关系：注册成功后跳转登录页，登录页可跳转忘记密码页

**2. 主流程入口（1页）**
- 首页（包含导航栏和页脚，作为所有页面的基础布局）

**3. 商品浏览流程（3页）**
- 商品列表页 → 商品详情页 → 店铺商品列表页
- 关系：首页/商品列表页 → 商品详情页 → 店铺商品列表页

**4. 购物流程（3页）**
- 购物车页 → 订单确认页 → 支付页面
- 关系：商品详情页/购物车页 → 订单确认页 → 支付页面

**5. 订单管理流程（3页）**
- 订单列表页 → 订单详情页 → 申请退款页（弹窗/独立页）
- 关系：个人中心/导航栏 → 订单列表页 → 订单详情页 → 申请退款页

**6. 个人中心流程（6页）**
- 个人中心页 → 个人信息编辑页、修改密码页、收货地址管理页、添加/编辑地址页、优惠券列表页
- 关系：个人中心页作为入口，链接到各个子页面

### 页面清单（按设计顺序）

1. **注册页** - 手机号/邮箱注册，验证码验证
2. **登录页** - 密码登录/短信验证码登录
3. **忘记密码页** - 验证码重置密码
4. **首页** - 商品分类导航、热门商品、轮播图（包含导航栏和页脚模板）
5. **商品列表页** - 商品卡片展示、分类筛选、排序、分页
6. **商品详情页** - 商品信息、图片、库存、加入购物车/立即购买
7. **店铺商品列表页** - 店铺信息、该店铺商品列表
8. **购物车页** - 购物车商品列表、数量修改、结算
9. **订单确认页** - 订单商品信息、地址选择、优惠券选择、支付方式
10. **支付页面** - 订单信息、支付二维码、支付状态
11. **个人中心页** - 个人信息、功能入口导航
12. **订单列表页** - 用户所有订单列表、状态筛选
13. **订单详情页** - 订单完整信息、订单操作按钮
14. **申请退款页** - 退款原因选择、退款说明
15. **个人信息编辑页** - 昵称、头像、手机号、邮箱编辑
16. **修改密码页** - 原密码、新密码、验证码
17. **收货地址管理页** - 地址列表、添加/编辑/删除
18. **添加/编辑地址页** - 地址表单（弹窗或独立页）
19. **优惠券列表页** - 优惠券列表展示（可选，或在个人中心展示）

**特殊页面**：
- **申请成为商家页** - 商家申请表单（用户端功能，但属于商家流程入口）

## 阶段二：商家端界面设计（9个页面）

### 设计复用说明

商家端界面**复用用户端的设计模式**，但功能聚焦在管理：

**复用关系**：
- 商家后台首页 → 复用首页的导航栏布局
- 商品管理页 → 复用商品列表页的布局，添加管理操作按钮
- 商品编辑页 → 复用表单页面布局
- 订单管理页 → 复用订单列表页布局，按店铺筛选
- 订单详情页 → 复用用户端订单详情页布局
- 销售报表页 → 复用数据展示页面布局

### 页面清单（按设计顺序）

1. **商家后台首页** - 导航菜单、用户信息（复用首页导航栏设计）
2. **店铺管理页** - 店铺列表、创建/编辑/删除店铺（复用列表页布局）
3. **店铺编辑页** - 店铺信息表单（复用表单页面布局）
4. **商品管理页** - 店铺选择器、商品列表、管理操作（复用商品列表页布局）
5. **商品编辑页** - 商品信息表单、店铺选择（复用表单页面布局）
6. **订单管理页** - 店铺选择器、订单列表（复用订单列表页布局）
7. **订单详情页** - 订单信息、退款审核（复用订单详情页布局）
8. **销售报表页** - 统计数据、趋势图表（复用数据展示布局）
9. **操作审计日志页** - 日志列表、筛选功能（复用列表页布局）

---

## 阶段三：管理员端界面设计（8个页面）

### 设计复用说明

管理员端界面**复用商家端的设计模式**，但扩展为全平台管理：

**复用关系**：
- 管理后台首页 → 复用商家后台首页布局
- 商家管理页 → 复用店铺管理页布局，添加审核功能
- 店铺管理页 → 复用商家端店铺管理页布局，添加审核功能
- 商品分类管理页 → 复用列表页布局
- 订单管理页 → 复用商家端订单管理页布局，显示所有订单
- 销售报表页 → 复用商家端报表页布局，显示全平台数据
- 审计日志页 → 复用商家端审计日志页布局，显示所有日志

### 页面清单（按设计顺序）

1. **管理后台首页** - 导航菜单、用户信息（复用商家后台首页）
2. **商家管理页** - 商家列表、审核功能、状态管理（复用店铺管理页布局）
3. **店铺管理页** - 所有店铺列表、审核功能、状态管理（复用商家端店铺管理页）
4. **商品分类管理页** - 分类树、增删改查（复用列表页布局）
5. **订单管理页** - 所有订单列表、退款审核（复用商家端订单管理页）
6. **订单详情页** - 订单信息、退款审核（复用商家端订单详情页）
7. **销售报表页** - 全平台统计数据、趋势图表（复用商家端报表页）
8. **操作审计日志页** - 所有日志列表、筛选功能（复用商家端审计日志页）

## 设计依赖关系图

```
阶段一：用户端（基础）
├── 认证流程（注册→登录→忘记密码）
├── 首页（基础布局模板）
├── 商品浏览（列表→详情→店铺列表）
├── 购物流程（购物车→订单确认→支付）
├── 订单管理（列表→详情→退款）
└── 个人中心（中心页→各子页面）

阶段二：商家端（复用阶段一）
├── 后台首页（复用首页导航栏）
├── 店铺管理（复用列表页布局）
├── 商品管理（复用商品列表页布局）
├── 订单管理（复用订单列表页布局）
└── 报表/日志（复用数据展示布局）

阶段三：管理员端（复用阶段二）
├── 后台首页（复用商家后台首页）
├── 商家/店铺管理（复用商家端管理页，添加审核）
├── 订单管理（复用商家端订单页，显示全部）
└── 报表/日志（复用商家端报表页，显示全平台）
```

---

## 关键设计要点

1. **导航栏复用**：所有页面使用统一的顶部导航栏和底部页脚
2. **列表页复用**：商品列表、订单列表、店铺列表使用相同的卡片布局模式
3. **表单页复用**：所有编辑/添加页面使用统一的表单布局
4. **详情页复用**：订单详情、商品详情使用相同的信息展示模式
5. **审核功能扩展**：管理员页面在商家页面基础上添加审核按钮和状态管理模块
