# Visily AI 界面设计 Prompt

## 阶段一 Prompt：用户端基础界面设计

```
你是一位专业的UI/UX设计师，需要为电商系统设计用户端界面。请按照以下要求设计19个页面。

### 项目背景
这是一个基于Vue 3 + daisyUI的电商系统，需要设计用户购物流程的所有界面。

### 配色方案
**整体策略**：中性色调为主，低饱和度背景，强调色使用≤10%

**浅色主题**：
- 基础背景： #FFFFFF
- 次要背景/卡片： #F3F4F6
- 主文本： #111827
- 次要文本： #6B7280

**深色主题**：
- 基础背景： #0F172A
- 卡片背景： #1E293B
- 主文本： #E5E7EB
- 次要文本：灰蓝色调

**强调色**：
- 强调色A（金色-主要CTA）：浅色模式 #EAB308，深色模式 #D4AF37
  - 用途：购买按钮、价格、折扣标签、VIP标签
- 强调色B（靛蓝-次要操作）：#6366F1
  - 用途：次要按钮、文本链接、非关键交互元素

**功能色**：
- 成功：绿色调（柔和，非霓虹）
- 警告：琥珀色调
- 错误：柔和红色（避免纯红）
- 信息：靛蓝/蓝色

### 设计原则
1. 现代、简洁、中性、高端、可信、转化导向、极简、电商友好、深色模式友好
2. 遵循daisyUI组件库设计规范
3. 响应式设计，桌面端为主，移动端基本适配
4. 支持中文界面
5. 确保WCAG AA或以上对比度
6. 避免纯黑纯白对比，避免高饱和度或霓虹色

### 页面设计顺序（19页）

**第一组：认证流程（3页）**
1. **注册页**
   - 手机号/邮箱注册（二选一）
   - 验证码输入（6位数字）
   - 密码设置（至少8位，包含字母和数字）
   - 获取验证码按钮（带倒计时）
   - 错误提示区域

2. **登录页**
   - 登录方式切换：密码登录 / 短信验证码登录
   - 手机号/邮箱输入
   - 密码输入（密码登录模式）
   - 验证码输入（短信验证码登录模式）
   - 获取验证码按钮
   - 忘记密码链接
   - 注册链接

3. **忘记密码页**
   - 手机号/邮箱输入
   - 验证码输入
   - 新密码输入
   - 确认密码输入
   - 获取验证码按钮
   - 重置密码按钮

**第二组：主入口（1页）**
4. **首页**
   - 顶部导航栏：Logo、搜索框（可选）、用户信息、购物车图标、登录/注册入口
   - 轮播图区域
   - 商品分类导航（横向或纵向）
   - 热门商品展示（卡片网格布局）
   - 底部页脚：版权信息、联系方式
   - 注意：此页面的导航栏和页脚将作为所有页面的基础模板

**第三组：商品浏览（3页）**
5. **商品列表页**
   - 复用首页导航栏和页脚
   - 分类筛选侧边栏或顶部标签
   - 排序选择器（价格、销量、上架时间）
   - 商品卡片网格（每行3-4个）：
     - 商品图片
     - 商品名称
     - 价格（原价灰色删除线，现价金色强调）
     - 库存状态
     - 店铺信息（可点击）
   - 分页组件

6. **商品详情页**
   - 复用导航栏和页脚
   - 左侧：商品图片轮播/放大
   - 右侧：商品信息
     - 商品名称
     - 价格（原价灰色删除线，现价金色大号字体）
     - 库存显示（≤50时显示预警提示）
     - 商品描述
     - 商品分类
     - 店铺信息（可点击跳转）
     - 数量选择器
     - "加入购物车"按钮（金色强调）
     - "立即购买"按钮（金色强调）
   - 底部：商品详情标签页

7. **店铺商品列表页**
   - 复用导航栏和页脚
   - 顶部：店铺信息卡片（店铺名称、描述）
   - 商品列表（复用商品列表页布局）
   - 分类筛选、排序、分页

**第四组：购物流程（3页）**
8. **购物车页**
   - 复用导航栏和页脚
   - 购物车商品列表（表格或卡片）：
     - 商品图片、名称、价格
     - 数量选择器（+/-按钮）
     - 删除按钮
     - 小计金额
   - 底部：总金额显示、结算按钮（金色强调）
   - 空购物车状态提示

9. **订单确认页**
   - 复用导航栏和页脚
   - 订单商品信息（按店铺分组显示）
   - 收货地址选择（下拉或卡片选择，可添加新地址）
   - 优惠券选择（下拉选择，显示可用优惠券）
   - 订单金额明细：
     - 商品总价
     - 优惠金额（灰色）
     - 实付金额（金色大号字体）
   - 支付方式选择（微信/支付宝）
   - "提交订单"按钮（金色强调）

10. **支付页面**
    - 订单信息卡片（订单号、金额）
    - 支付方式选择
    - 支付二维码展示区域（或跳转提示）
    - 支付状态提示（支付中、成功、失败）
    - 倒计时提示（30分钟未支付自动取消）
    - 返回按钮

**第五组：订单管理（3页）**
11. **订单列表页**
    - 复用导航栏和页脚
    - 订单状态筛选标签（待付款、已付款、待发货等）
    - 订单卡片列表：
      - 订单号、创建时间
      - 商品缩略图（多个商品横向展示）
      - 订单状态标签
      - 订单金额
      - 操作按钮（根据状态显示：取消、确认收货、申请退款等）
    - 空状态提示

12. **订单详情页**
    - 复用导航栏和页脚
    - 订单信息卡片：
      - 订单号、状态、创建时间
      - 商品列表（商品快照信息）
      - 收货地址信息
      - 订单金额明细
      - 支付信息
    - 订单操作按钮区域（根据状态显示）
    - 退款信息展示（如有）

13. **申请退款页**（弹窗或独立页）
    - 退款原因选择（下拉或单选）
    - 退款说明输入框（可选）
    - 提交按钮、取消按钮

**第六组：个人中心（6页）**
14. **个人中心页**
    - 复用导航栏和页脚
    - 个人信息卡片（头像、昵称、手机号、邮箱）
    - 功能入口网格：
      - 订单管理
      - 收货地址管理
      - 优惠券
      - 个人信息编辑
      - 修改密码
      - 申请成为商家（如果还不是商家）
      - 商家后台（如果是商家）
    - 退出登录按钮

15. **个人信息编辑页**
    - 复用导航栏和页脚
    - 表单：
      - 头像上传/显示
      - 昵称输入
      - 手机号显示（如需修改需验证）
      - 邮箱显示（如需修改需验证）
    - 保存按钮、取消按钮

16. **修改密码页**
    - 复用导航栏和页脚
    - 表单：
      - 原密码输入
      - 新密码输入（密码强度提示）
      - 确认密码输入
      - 获取验证码按钮
      - 验证码输入
    - 修改按钮、取消按钮

17. **收货地址管理页**
    - 复用导航栏和页脚
    - 地址列表卡片：
      - 收货人、电话、详细地址
      - 默认地址标签
      - 编辑按钮、删除按钮
    - 添加地址按钮
    - 空状态提示

18. **添加/编辑地址页**（弹窗或独立页）
    - 表单：
      - 收货人姓名
      - 联系电话
      - 详细地址
      - 设为默认地址复选框
    - 保存按钮、取消按钮

19. **优惠券列表页**（可选）
    - 复用导航栏和页脚
    - 优惠券卡片列表（可用/已用状态区分）
    - 优惠券信息：类型、折扣、数量、有效期

**特殊页面**：
20. **申请成为商家页**
    - 复用导航栏和页脚
    - 表单：
      - 店铺名称
      - 店铺描述
    - 提交按钮、取消按钮

### 设计要求
- 所有页面保持统一的视觉风格
- 按钮使用金色（#EAB308）作为主要CTA，靛蓝（#6366F1）作为次要操作
- 价格信息使用金色强调
- 卡片使用浅灰色背景（#F3F4F6），圆角设计
- 表单输入框统一样式，错误状态使用柔和红色
- 加载状态使用统一的loading动画
- 错误/成功提示使用统一的toast或alert样式

请按照以上顺序逐一设计每个页面，确保页面间的视觉一致性和交互流畅性。
```

## 阶段二 Prompt：商家端管理界面设计

```
你是一位专业的UI/UX设计师，需要为电商系统设计商家端管理界面。本阶段基于阶段一的设计，复用用户端的布局模式和视觉风格，但功能聚焦在商家管理。

### 项目背景
商家端界面是商家管理自己店铺、商品、订单的后台系统。需要复用阶段一的设计模式，但调整为管理视角。

### 配色方案
与阶段一完全相同：
- 浅色主题：基础背景 #FFFFFF，卡片 #F3F4F6，主文本 #111827
- 深色主题：基础背景 #0F172A，卡片 #1E293B，主文本 #E5E7EB
- 强调色A（金色）：#EAB308（主要CTA）
- 强调色B（靛蓝）：#6366F1（次要操作）

### 设计复用原则
1. **导航栏复用**：使用与用户端首页相同的导航栏布局，但菜单项改为商家管理功能
2. **列表页复用**：商品列表、订单列表复用用户端的卡片网格布局
3. **表单页复用**：编辑页面复用用户端的表单布局样式
4. **详情页复用**：订单详情复用用户端订单详情页的信息展示模式
5. **新增管理功能**：在复用布局基础上，添加管理操作按钮（编辑、删除、状态切换等）

### 页面设计顺序（9页）

1. **商家后台首页**
   - 复用用户端首页的导航栏设计
   - 左侧或顶部导航菜单：
     - 店铺管理
     - 商品管理
     - 订单管理
     - 销售报表
     - 操作日志
   - 右侧：用户信息、退出登录
   - 主内容区：数据概览卡片（可选）

2. **店铺管理页**
   - 复用用户端列表页的布局
   - 顶部：创建店铺按钮（金色强调，检查最多3个店铺限制）
   - 店铺卡片列表（复用商品卡片布局）：
     - 店铺名称
     - 店铺描述
     - 状态标签（正常/禁用）
     - 操作按钮：编辑、删除、状态切换
   - 空状态提示

3. **店铺编辑页**（添加/编辑共用）
   - 复用用户端表单页布局
   - 表单：
     - 店铺名称（必填）
     - 店铺描述（可选）
   - 保存按钮（金色强调）、取消按钮

4. **商品管理页**
   - 复用用户端商品列表页布局
   - 顶部：
     - 店铺选择器（下拉选择，必选）
     - 添加商品按钮（金色强调）
   - 商品列表（复用商品卡片网格）：
     - 商品图片、名称、价格、库存
     - 状态标签（上架/下架）
     - 库存预警提示（≤50时高亮）
     - 操作按钮：编辑、删除、上架/下架切换
   - 分类筛选、搜索、分页

5. **商品编辑页**（添加/编辑共用）
   - 复用用户端表单页布局
   - 表单：
     - 店铺选择（必选，下拉）
     - 商品名称（必填）
     - 商品价格（必填，数字输入）
     - 商品库存（必填，数字输入）
     - 商品分类（必填，下拉，支持创建新分类）
     - 商品描述（可选，多行文本）
     - 商品图片URL（必填，格式验证）
     - 上架/下架状态选择
   - 保存按钮（金色强调）、取消按钮

6. **订单管理页**
   - 复用用户端订单列表页布局
   - 顶部：店铺选择器（下拉选择）
   - 订单列表（复用订单卡片布局）：
     - 订单号、创建时间
     - 商品缩略图
     - 订单状态标签
     - 订单金额
     - 操作按钮：查看详情、退款审核入口
   - 状态筛选、分页

7. **订单详情页**（商家端）
   - 复用用户端订单详情页布局
   - 订单信息展示（仅显示该商家店铺的订单项）
   - 退款审核功能区域（如有退款申请）：
     - 退款原因、退款说明
     - 同意/拒绝按钮（金色/红色）
     - 拒绝原因输入框（拒绝时显示）

8. **销售报表页**
   - 复用数据展示页面布局
   - 顶部：店铺选择器
   - 统计数据卡片（3个）：
     - 总销售额（金色大号数字）
     - 总订单数
     - 总商品销量
   - 趋势图表区域（折线图）：
     - 销售额趋势
     - 订单数趋势
     - 商品销量趋势
   - 日期范围选择器

9. **操作审计日志页**
   - 复用列表页布局
   - 日志列表（表格或卡片）：
     - 操作时间
     - 操作类型
     - 操作对象
     - 操作内容
     - 操作结果
   - 操作类型筛选、时间范围筛选、分页

### 设计要求
- 所有页面复用阶段一的视觉风格和组件样式
- 管理操作按钮使用统一的图标和颜色（编辑：靛蓝，删除：红色，状态切换：金色）
- 表格/列表使用浅灰色背景卡片，hover效果
- 表单验证错误提示使用统一的样式
- 数据统计卡片使用金色强调数字
- 图表使用统一的配色方案（金色、靛蓝、灰色）

请确保商家端界面与用户端界面保持视觉一致性，同时突出管理功能的专业性。
```

## 阶段三 Prompt：管理员端审核界面设计

```
你是一位专业的UI/UX设计师，需要为电商系统设计管理员端界面。本阶段基于阶段二的设计，复用商家端的管理界面布局，但扩展为全平台管理，并添加审核功能。

### 项目背景
管理员端界面是系统管理员管理全平台商家、店铺、订单的后台系统。需要复用阶段二的设计模式，但扩展为全平台视角，并添加审核功能模块。

### 配色方案
与阶段一、二完全相同：
- 浅色主题：基础背景 #FFFFFF，卡片 #F3F4F6，主文本 #111827
- 深色主题：基础背景 #0F172A，卡片 #1E293B，主文本 #E5E7EB
- 强调色A（金色）：#EAB308（主要CTA）
- 强调色B（靛蓝）：#6366F1（次要操作）

### 设计复用原则
1. **导航栏复用**：使用与商家端相同的导航栏布局，菜单项改为管理员功能
2. **管理页复用**：商家管理、店铺管理复用商家端店铺管理页布局
3. **列表页复用**：订单管理、审计日志复用商家端列表页布局
4. **报表页复用**：销售报表复用商家端报表页布局，但显示全平台数据
5. **新增审核功能**：在复用布局基础上，添加审核按钮、审核开关配置、待审核列表等模块

### 页面设计顺序（8页）

1. **管理后台首页**
   - 复用商家后台首页布局
   - 左侧或顶部导航菜单：
     - 商家管理
     - 店铺管理
     - 商品分类管理
     - 订单管理
     - 销售报表
     - 操作日志
   - 右侧：用户信息、退出登录
   - 主内容区：平台数据概览卡片（可选）

2. **商家管理页**
   - 复用商家端店铺管理页布局
   - 顶部：
     - 商家审核开关配置（切换按钮：开启审核/不审核）
     - 待审核商家列表标签（如果开启了审核）
   - 商家列表（复用卡片布局）：
     - 商家名称、负责人、注册时间
     - 状态标签（正常/禁用/待审核）
     - 操作按钮：
       - 审核按钮（待审核状态时显示：同意/拒绝，金色/红色）
       - 状态管理（启用/禁用切换）
   - 搜索功能

3. **店铺管理页**
   - 复用商家端店铺管理页布局
   - 顶部：
     - 店铺审核开关配置（切换按钮：开启审核/不审核）
```
